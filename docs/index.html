Marionette

A desktop coding agent built on the Puppetmaster orchestrator.

Most coding agents are a single model in a chat loop. Marionette takes a different shape: it sits on top of Puppetmaster, an orchestration engine that splits work into parallel workers and routes each task to the cheapest model that can actually handle it. Trivial work runs on cheap or local models. Hard problems get a frontier model. You are not locked to any one provider.

Every worker is a real subprocess with durable state in SQLite, so runs are replayable and auditable after the fact. The agent works over a live CodeGraph of your repository instead of grepping around, and it can edit and update its own source. The model is a swappable part. The orchestration is the point.

Install from the terminal

curl -fsSL https://professorpalmer.github.io/marionette/install.sh | bash

Updates are continuous. The in-app update pill pulls main, rebuilds, and relaunches, so a merge reaches every install on its next click.

What you get

  • Worker swarms. Fan work out to many workers running in parallel, each with its own durable, inspectable state. Not one long conversation.
  • Cost-aware model routing. A tiered router picks the model per task. You can watch what each job cost and why that model was chosen.
  • CodeGraph retrieval. Queries against an indexed graph of your repo instead of repeated directory crawls.
  • A personal knowledge wiki. Sessions distill into a portable, cross-session wiki you own as plain markdown.
  • Runs from source. The same code on macOS, Windows, and Linux. stdlib-only Python backend; Puppetmaster is the one real dependency.
